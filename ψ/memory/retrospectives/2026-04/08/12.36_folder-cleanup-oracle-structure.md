## Honest Feedback

**3 Friction Points**:

1. **`ghq get` behavior ไม่ชัดเจน**: `ghq get` บน repo ที่ clone มาแล้วจะ update in-place แต่บน path ที่ไม่มีใน ghq จะ clone ใหม่ทำให้เกิด duplicate ที่ไม่ได้ตั้งใจ ควรมี convention ชัดว่า Oracle repos ไม่ใช้ `ghq get` แต่ใช้ symlink approach

2. **start-oracle.sh แก้ 2 รอบ**: ครั้งแรกแก้ path เป็น arra-oracle-v3/frontend แต่ผิดเพราะ frontend คือ oracle-studio ซึ่งเป็น separate repo ถ้า check README ก่อนก็แก้ถูกรอบเดียว ควรอ่าน README ก่อนเสมอเมื่อแก้ script ที่ชี้ไป external repos

3. **writer-oracle ยังไม่ได้ตรวจ**: session นี้ข้ามไป writer-oracle โดยไม่ได้ตรวจว่าอยู่ถูกที่ไหมหรือมี context อะไรบ้าง อาจจะเป็น Oracle ที่เตรียมสร้างหรือ stale copy ก็ได้ ควร check ใน session ต่อไป

## Lessons Learned

1. **`ghq get` ไม่ใช่ symlink** — มัน clone fresh จาก GitHub เสมอ ถ้าจะให้ maw fleet ชี้ไป existing repo ใช้ `ln -sf` แทน
2. **~/repos = Oracle repos ของ Mieng เท่านั้น** — tool repos (maw-js, arra-oracle-v3, oracle-studio) ควรอยู่ใน ghq/github.com/Soul-Brews-Studio/
3. **อ่าน README ก่อนแก้ path ใน scripts** — โดยเฉพาะ external repos ที่อาจมี separate frontend/backend
